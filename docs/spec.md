# Offline LLM Eval 公開仕様

## 目的

Offline LLM Eval は、RAG/LLM 機能の評価ケースを固定し、回答品質と運用エラーを同じ形式で確認するためのワークベンチです。

v0.1.0 は外部APIに依存しない評価基盤として、評価ケースと実行結果を扱う DB 構造、判定ロジック、既存 run を参照する API、確認結果を反映する API、既存 run に対する品質判定 CLI、検証証跡Markdown出力を扱います。

## 提供する範囲

1. 評価ケース、実行結果、判定結果を扱う DB schema（テーブル定義）を用意する。
2. mock provider（固定応答を返す実行部品）と判定ロジックを、`src/` 配下の関数・クラスとして提供する。
3. 既存 run の一覧、詳細、ケース詳細、差分、検証証跡を API から参照する。
4. 確認待ちケースを確認し、確認結果を API から反映する。
5. 既存 run を品質判定設定に基づいて確認し、CI で扱える終了コードと判定結果を返す。
6. 実行ごとの検証証跡を Markdown として出力する。

## このバージョンで提供しない入口

- データセット取り込みから mock provider での実行、判定、評価結果保存までを一括で起動する CLI/API
- 外部LLMサービスへ接続して回答を取得する provider

## 主張範囲

- fixture（再現用の固定データ）を想定した評価ケースと判定結果の管理
- 引用整合性、回答しないべきケース、応答形式、応答時間、外部サービス相当のエラーを判定するロジック
- secret pattern（secret値に似た文字列パターン）検出と、secret値を伏せた検証証跡出力
- SQLite 既定接続と PostgreSQL 接続URLを受け付ける DB 接続構成。v0.1.0 の自動検証は SQLite 接続を対象にする。
- 既存 run を扱う localhost 向け API と CLI 操作

## 主張しない範囲

- 運用環境でそのまま使えること
- あらゆるLLM出力品質を評価できること
- 外部LLMサービスの費用を制御できること
- 長時間運用に耐えること
- 外部LLMサービスの品質評価
- 認証、認可、監視、障害対応を含む運用一式
- データセット取り込みから評価実行と結果保存までを一括で起動する CLI/API

## 検証証跡Markdown

検証証跡Markdownは、実行情報、ケース集計、失敗ケース、注意が必要なケース、secret値に似た文字列の検出結果、`not_claimed`（主張しない範囲）を含みます。

`not_claimed` は、その検証証跡で主張しない範囲を明示するための必須セクションです。既定値は次の通りです。

- 運用環境でそのまま使えること
- あらゆるLLM出力品質を評価できること
- 外部LLMサービスの費用を制御できること
- 長時間運用に耐えること

secret値は、検証証跡の出力前にカテゴリ付きの伏せ字表現へ置き換えます。

## 検証方法

主要な検証は pytest、ruff、mypy で行います。

```powershell
python -m pytest tests
python -m ruff check alembic src tests
python -m ruff format --check alembic src tests
python -m mypy src tests
```
