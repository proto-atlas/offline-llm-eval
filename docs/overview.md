# Offline LLM Eval 概要

## 目的

Offline LLM Eval は、RAG/LLM 機能の回答品質を、固定した評価ケースで継続的に確認するためのワークベンチです。

v0.1.0 では、評価ケースと実行結果を扱う DB 構造、判定ロジック、既存 run を参照する API、確認結果を反映する API、既存 run に対する品質判定 CLI を提供します。

## 扱う範囲

- 評価ケース、実行結果、判定結果を扱う DB 構造
- 引用整合性、回答しないべきケース、応答JSON形式、応答時間、外部サービス相当のエラーを判定するロジック
- 既存 run の一覧、詳細、ケース詳細、差分、検証証跡を返す API
- 既存 run を対象にした品質判定 CLI
- 確認待ちケースと確認結果の反映
- secret値を伏せた検証証跡Markdownの生成

## 扱わない範囲

- 本番環境向けの認証/認可
- 長時間稼働の評価
- あらゆるLLM出力品質の評価
- 外部LLMサービスの品質や費用の評価
- 監視、障害対応、SLA を含む運用一式
- データセット取り込みから評価実行と結果保存までを一括で起動する CLI/API

## 設計方針

- 評価ケースは入力、期待条件、参照データ、判定結果を分けて扱う。
- 自動判定で扱える条件と、人間のレビューが必要な条件を分ける。
- 外部サービス相当のエラーや応答形式違反は、回答品質の低下とは別カテゴリで保存する。
- 基準実行との差分比較は、同じ評価ケースを同じ条件で再実行できることを前提にする。
- 検証証跡Markdownには `not_claimed`（主張しない範囲）を含め、検証結果から主張しない範囲を明示する。
